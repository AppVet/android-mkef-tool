Step 1. Create AppVet Tool Account

In an AppVet system, online tools are required to have a user account on AppVet in order to log in and post reports and risk assessments for an app. To add an AppVet user account for the MasterKey-ExtraField Scanner tool, start AppVet (i.e., start Tomcat on the AppVet server) and log in using an ADMIN user account. Next, select Admin > Users and select the Add button. For this tool, set User ID to 'mkef' and set Role to 'TOOL_PROVIDER' in the Add User dialog box. All other parameters should be set to appropriate values as shown, for example, in Figure 1. Remember your selected Password as this will be needed later to configure the APK Signature Verifier.




Figure 1. Adding Signature Verifier Tool Account.



After adding user 'sigverifier', select Submit and shutdown AppVet (i.e., shutdown Tomcat).

Step 2. Install AppVet Tool Adapter

From the AppVet server, open a browser and download the latest APK Signature Verifier tool adapter android-sigverifier-tool.xml file at:

https://github.com/AppVet/android-sigverifier-tool/releases


and move it to your $APPVET_FILES_HOME/conf/tool_adapters/android directory.

Next, edit android-sigverifier-tool.xml by changing the <appvet:URL> property to reflect the tool server's hostname or IP address. If the APK Signature Verifier resides on the same server as AppVet, then set <appvet:URL> to:

		
	<appvet:URL>http://127.0.0.1:8080/android-sigverifier-tool/Service</appvet:URL>

If, however, APK Signature Verifier resides on a different server (e.g., tools.example.com) and uses SSL and port 8443, then set <appvet:URL> to: 

		
	<appvet:URL>https://tools.example.com:8443/android-sigverifier-tool/Service</appvet:URL>

Note that the '/android-sigverifier-tool/Service' portion of the URL should never be modified. For more information on Tool Adapter Properties, see Tool Adapter Properties.

Step 3. Test or Install Tool

Test

If you are connecting to an existing APK Signature Verifier tool that is accessible to AppVet, you can test the tool now by logging into AppVet, uploading an app, and viewing the APK Signature Verifier report. The APK Signature Verifier status and report are shown in Figure 1.




Figure 1. APK Signature Verifier.




If you experience errors connecting to the tool, verify the tool adapter properties or contact the tool's host administrator.

Install Tool

If you do not yet have access to an existing APK Signature Verifier tool, you must install it. For more information, see the APK Signature Verifier Tool Installation.
