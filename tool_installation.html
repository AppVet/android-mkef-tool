Tool Installation

Installation of the APK Signature Verifier tool requires the following steps:

  • Step 1: Install Dependencies
  • Step 2: Download and Install APK Signature Verifier
  • Step 3: Set Environment Variables
  • Step 4: Configure
  • Step 5: Test

Step 1: Install Dependencies

APK Signature Verifier requires the installation of Oracle JDK and Apache Tomcat. If APK Signature Verifier is installed on the same server as AppVet, then Oracle JDK and Apache Tomcat should already be installed. Otherwise, to install Oracle JDK, see the instructions for Installing Oracle JDK. To install Apache Tomcat, see the instructions for Installing Apache Tomcat.

Step 2. Download and Install APK Signature Verifier

Download

  • Step a: Download the latest APK Signature Verifier binary release android-sigverifier-tool-1.0-binary.tar.gz file from https://github.com/AppVet/android-sigverifier-tool/releases.
  • Step b: Extract the compressed file onto your local machine (e.g., using tar -zxvf android-sigverifier-tool-1.0-binary.tar.gz)

Install

Before installing APK Signature Verifier, ensure that Apache Tomcat is shutdown. Then, from the extracted APK Signature Verifier binary release:
  • Step a: Copy the android-sigverifier-tool.war file to $CATALINA_HOME/webapps.
  • Step b: Copy the android_sigverifier_files directory to your $HOME (i.e., /home/<username>) directory.

Step 3. Set Environment Variable

To ensure proper operation of APK Signature Verifier, set a global environment variable $ANDROID_SIGVERIFIER_FILES_HOME to the installation path of your android_sigverifier_files directory. For example, if android_sigverifier_files was copied to your $HOME directory , set:

$ANDROID_SIGVERIFIER_FILES_HOME=/home/<username>/android_sigverifier_files

To ensure the proper setting of global environment variables under CentOS/RedHat, see Setting Global Environment Variables.

Step 4. Configure

Edit the $ANDROID_SIGVERIFIER_FILES_HOME/conf/ToolProperties.xml file as follows:

ServiceURL

Set the <appvet:ServiceURL> property that defines the URL of the APK Signature Verifier tool that should be used by AppVet to initiate a connection. If APK Signature Verifier is installed on the same server as AppVet, then set <appvet:ServiceURL> to: 

		
   <appvet:ServiceURL>http://127.0.0.1:8080/android-sigverifier-tool</appvet:ServiceURL>

If, however, APK Signature Verifier is installed on a different server than AppVet (e.g., tools.example.com ) using SSL and port 443, set <appvet:ServiceURL> to: 

		
   <appvet:ServiceURL>https://tools.example.com:8443/android-sigverifier-tool</appvet:ServiceURL>

Note that the ' /android-sigverifier-tool ' portion of the URL should never be modified.

AppVet

Set the <appvet:AppVet> property that defines the AppVet login information. Here, only two parameters should be set: <appvet:URL> and <appvet:Password> . Set the <appvet:URL> to the URL of the AppVet server. If AppVet resides on the same server, set <appvet:URL> to: 

		
   <appvet:URL>http://127.0.0.1:8080/appvet/AppVetServlet</appvet:URL>

If AppVet resides on a different server (e.g., server.mytest.org ) using SSL and port 8443, set <appvet:URL> to: 

		
   <appvet:URL>https://server.mytest.org:8443/appvet/AppVetServlet</appvet:URL>

Note that the '/appvet/AppVetServlet' portion of the URL should never be modified.

Next, replace 'sigverifier-password' with the AppVet password for your APK Signature Verifier tool in the <appvet:Password> property: 

		
   <appvet:Password>sigverifier-password</appvet:Password>

For more information on Tool Properties, see Tool Properties.

Step 5. Test

To test the proper installation of APK Signature Verifier, start the tool by starting Apache Tomcat. If AppVet resides on the same server, starting Tomcat will start both AppVet and the tool. If AppVet resides on a different server, start Tomcat on that server. To start Tomcat, run:

# $HOME/apache-tomcat-8.<version>/bin/startup.sh



On the server hosting the tool, open a web browser with the following URL:

http/s://<tool_server_host>:<port>/android-sigverifier-tool/index.html



If the message 'APK Signature Verifier is running.' then APK Signature Verifier was installed.

Next, log into AppVet, upload an app, and view the APK Signature Verifier report. The APK Signature Verifier status and report are shown in Figure 1.
